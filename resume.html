A Graphics Library

In university I became interested in non-square pixels, for instance using grids of triangles and hexagons.
That led to the idea of using arbitrary plane tesselations, particularly ones composed of regular polygons (for instance the Archimedean tilings).
The ultimate goal is to make computer games whose graphics and physics engines are based on such geometric spaces.
The implementations are usually called "geom", and are usually the first thing I try to write when I'm designing or implementing a new programming language.
The current iteration is a C library, to be used as a shared library by a higher-level language.
